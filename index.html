Markdown

is a pretty good intermediary

  1. It has a fairly simple syntax.
  2. It compiles into clean HTML
  3. With the right add-ons, it converts to all document formats: .doc, .pdf, .rtf, html (of course), and more.


The idea is that Markdown's formatting syntax is to make it as readable as possible. A Markdown-formated document could be publishable as-is, as plain text, and still communicate structure.

Getting Started

You'll need ...

  1. Something to write markdown in.
    • Plenty of apps for Mac, fewer for Windows
    • I like Sublime Text because there are a plethora of plugins, including Pandoc universal format conversion - which, while great on its own, kind of sucks if you don't like the command line.
    • Keep your eye on Editorially. Currently in beta, has a beautiful UI, and they will eventually support conversion to Word, PDF, etc.
    • For this workshop we'll use Dillinger.io
  2. That's about it.
